CLI

Browse and share
from the terminal.

The hub is built into the castor CLI. No account needed to browse. Share via GitHub PR.

Terminal
# Browse all recipes $ castor hub browse # Filter by category and AI provider $ castor hub browse --category agriculture --provider huggingface # Search for something specific $ castor hub search "outdoor patrol camera" # View full details $ castor hub show picar-home-patrol-e7f3a1 # Install a recipe to your project $ castor hub install picar-home-patrol-e7f3a1 # Share your working config (PII auto-scrubbed) $ castor hub share --submit # See all categories $ castor hub categories

Submit

How to submit a recipe

Recipes are submitted via GitHub Pull Request to the community-recipes/ directory.

1

Package your config

Run castor hub share --submit to auto-scrub secrets and generate a submission bundle.

Terminal
castor hub share --submit --config robot.rcan.yaml --docs BUILD_NOTES.md
2

Fork & PR

Fork the OpenCastor repo, add your bundle to community-recipes/, and open a Pull Request.

Terminal
git clone https://github.com/YOUR_USER/OpenCastor.git cp -r my-recipe/ community-recipes/ git add . && git commit -m "Add my-robot recipe" git push && # open PR on GitHub
3

Review & merge

Maintainers review for PII leaks and validate the RCAN schema. Once approved, your recipe appears in castor hub browse.

Safety

Share safely.
Secrets stay secret.

The castor hub share command automatically scrubs your config and docs before packaging. Always review the output before submitting.

üîë API Keys & Tokens

OpenAI, Anthropic, Google, Hugging Face, GitHub, Slack tokens are all detected and redacted.

üìß Personal Info

Email addresses, phone numbers, and hostnames are scrubbed automatically.

üåê Network Info

Public IPs are redacted. Private/local IPs (192.168.x.x, 10.x.x.x) are preserved since they're not unique.

üìÅ File Paths

Home directory paths (/home/username) are normalized to /home/user.
