From zero to robot
in five minutes.

Everything you need to install, configure, and deploy your first AI-powered robot.

Quick Start

1

Install OpenCastor

One command installs everything: Python deps, hardware drivers, and the interactive wizard.

curl -sL opencastor.com/install | bash
2

Run the setup wizard

The wizard walks you through choosing an AI provider, hardware kit, and messaging channels. It generates a validated RCAN config file and saves your API keys to .env. See full walkthrough below.

castor wizard
3

Launch

Start the runtime. Your robot is now an AI agent accepting commands via WhatsApp, voice, or API.

castor run --config my_robot.rcan.yaml

Setup Wizard

What to expect when
you run castor wizard

The wizard asks five questions, then generates a fully validated RCAN config file. Here is every step.

1

Name your robot

Give your project a name. This becomes the filename for your config and appears in mDNS discovery.

Terminal
Project Name [MyRobot]: Explorer
2

Choose your AI brain

Pick which large language model powers your robot. You can change this anytime by editing one line in your config file.

Terminal
--- BRAIN SELECTION --- Which AI provider do you want to use? [1] Anthropic Claude Opus 4.6 (Recommended) [2] Google Gemini 2.5 Flash [3] Google Gemini 3 Flash (Preview) [4] OpenAI GPT-4.1 [5] Hugging Face (Llama, Qwen, Mistral, etc.) -- 1M+ models [6] Local Llama (Ollama) -- free, runs on your machine Selection [1]: 1
Tip: Ollama (option 5) is completely free and runs locally on your Pi or laptop. No API key needed. Great for experimenting before committing to a paid provider.
3

Enter your API key

Paste your provider API key. The wizard saves it to a local .env file that is never committed to git. If you already have the key set in your environment, this step is skipped automatically.

Terminal
--- API KEY --- Your Anthropic Claude Opus 4.6 API key is needed. It will be saved to your local .env file (never committed to git). ANTHROPIC_API_KEY: sk-ant-... [OK] Saved to .env
4

Select your hardware

Pick a pre-built preset for popular Amazon robot kits, or choose Custom for advanced setups like Dynamixel arms. Presets configure motors, wiring, and physics automatically.

Terminal
--- HARDWARE KIT --- Select your hardware kit: [1] Custom (Advanced) [2] RPi RC Car + PCA9685 + CSI Camera (Recommended) [3] Waveshare AlphaBot ($45) [4] Adeept RaspTank ($55) [5] Freenove 4WD Car ($49) [6] SunFounder PiCar-X ($60) Selection [2]: 2
Custom mode walks you through defining degrees of freedom, link dimensions, and motor mappings. Use this for robot arms, hexapods, or any non-standard build.
5

Connect messaging channels

Optionally connect WhatsApp, Telegram, Discord, or Slack so you can command your robot from your phone. You can enable multiple channels at once or skip this entirely.

Terminal
--- MESSAGING CHANNELS --- Connect your robot to messaging platforms (optional). You can enable multiple channels. Enter numbers separated by commas. [0] None (skip) [1] WhatsApp (scan QR code) -- no account needed! [2] WhatsApp via Twilio (legacy) [3] Telegram Bot [4] Discord Bot [5] Slack Bot Selection (e.g. 1,3) [0]: 1
✓

Config generated

The wizard outputs a validated .rcan.yaml config file with all required sections: metadata, agent, physics, drivers, network, and RCAN protocol. It also auto-detects your robot's capabilities.

Terminal
Detected RCAN Capabilities: + status + nav + teleop + vision + chat ================================================== Setup Complete! Config file: explorer.rcan.yaml AI Provider: Anthropic Claude Opus 4.6 Model: claude-opus-4-6 Channels: WhatsApp (scan QR code) Next Steps: 1. Run the robot: castor run --config explorer.rcan.yaml 2. Start the gateway: castor gateway --config explorer.rcan.yaml 3. Open the dashboard: castor dashboard

What the wizard creates

explorer.rcan.yaml

Your robot config file. Contains AI provider, hardware definitions, physics model, RCAN protocol settings, and declared capabilities. Validates against the RCAN schema.

.env

Your API keys and channel credentials. Stored locally, listed in .gitignore, never committed. The runtime loads these automatically.

Minimal example

Everything you need to run your first AI-powered robot in a single config file.

my_robot.rcan.yaml
rcan_version: "1.0.0-alpha" metadata: robot_name: "Explorer" robot_uuid: "..." # auto-generated by wizard agent: provider: "anthropic" model: "claude-opus-4-6" vision_enabled: true safety_stop: true physics: type: "differential_drive" dof: 2 drivers: - protocol: "pca9685_i2c" address: "0x40" rcan_protocol: port: 8000 capabilities: [status, nav, teleop, chat]
